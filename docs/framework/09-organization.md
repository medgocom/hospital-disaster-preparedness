# 軸9: 組織体制（指揮命令系統・多職種連携）

## 概要

アンケート結果から、インフラのハード面だけでなく「組織としてどう動くか」への関心が非常に高いことが判明しました。本軸では、災害時の指揮命令系統、役割分担、多職種連携について整理します。

## リスク評価

### 組織体制の崩壊がもたらす影響

| 状況 | 影響 | 対応策 |
|-----|-----|-------|
| 指揮者不在 | 意思決定停滞 | 代行順位の明確化 |
| 役割不明確 | 重複・漏れ発生 | アクションカード |
| 情報途絶 | 状況把握不能 | クロノロジー・EMIS |
| 連携不足 | 患者対応遅延 | 多職種訓練 |

## 多職種連携のルール

### アンケートで挙がった課題

> 「誰がトリアージするのか」
> 「PT（理学療法士）でもトリアージの現場でトリアージできるのか」
> 「トリアージ時の連携」

### 職種別の役割定義

| 職種 | 災害時の主な役割 | トリアージ可否 |
|-----|----------------|--------------|
| 医師 | 治療判断、トリアージ統括 | ◎ 実施・最終判断 |
| 看護師 | トリアージ実施、患者ケア | ◎ 実施可能 |
| 救急救命士 | トリアージ実施、搬送 | ◎ 実施可能 |
| 薬剤師 | 医薬品管理、調剤 | △ 補助のみ |
| 臨床検査技師 | 検体検査、輸血対応 | △ 補助のみ |
| 放射線技師 | 画像検査 | △ 補助のみ |
| 理学療法士 | 搬送補助、リハ評価 | △ 研修修了者は可 |
| 事務職員 | 受付、情報管理、記録 | × 不可 |

### トリアージ実施のルール

1. **基本**: 医師・看護師・救急救命士が実施
2. **例外**: 研修を修了したコメディカルは「一次トリアージ」のみ可
3. **最終判断**: 必ず医師が確認・承認
4. **記録**: トリアージタグ + クロノロジーに記載

### EMIS（広域災害救急医療情報システム）

- **入力タイミング**: 災害発生後30分以内
- **入力担当者**: 情報班（事務職員）
- **入力内容**:
  - 病院の被災状況
  - 受入可能患者数
  - 必要な支援

### 情報伝達のルール

```
【報告の5W1H】
When:   いつ発生したか
Where:  どこで発生したか
What:   何が起きたか
Who:    誰が関係しているか
Why:    なぜ起きたか（分かれば）
How:    現在の状況・対応状況
```

## 空文化を防ぐ：現実的な参集計画と権限委譲

### なぜマニュアルは空文化するのか

27施設アンケートの結果、多くの病院が「全員参集」を前提としたBCPを策定していました。しかし現実には、以下の構造的問題により計画が空文化しています。

| 絶望ポイント | 現実 | マニュアル上の想定 |
|------------|------|------------------|
| 全員参集の嘘 | 夜間・休日は在院率10〜20%。交通寸断で参集できるのは近隣職員のみ | 「全職員が速やかに参集」 |
| 指揮者不在 | 院長・副院長が到着するまで数時間。当直医は判断権限を持たされていない | 「院長が本部長を務める」 |
| 人手不足の初動 | 夜勤3〜5名で全館の患者安全確認・設備点検を行う必要がある | 「各班が分担して対応」 |
| 運搬の限界 | エレベーター停止時、患者搬送は4名1組が必要。階段搬送は極めて困難 | 「必要に応じて搬送」 |

### 現実的な参集計画

#### 参集フェーズの定義

「全員参集」ではなく、段階的な人員確保を前提とする。

| フェーズ | 時間帯 | 在院想定人数 | 対応範囲 |
|---------|-------|------------|---------|
| Phase 0（発災直後） | 0〜15分 | 夜勤者のみ（3〜10名） | 患者安全確認・初期情報収集 |
| Phase 1（近隣参集） | 15〜60分 | +徒歩圏内職員（10〜30名） | 本部機能の立ち上げ・応急対応 |
| Phase 2（広域参集） | 1〜3時間 | +自動車圏内職員 | 各班の本格稼働 |
| Phase 3（全体参集） | 3時間〜 | 連絡がつく全職員 | 通常体制への移行 |

#### 参集区分の設定

全職員を居住地から施設までの距離に基づき区分する。

| 区分 | 基準 | 参集手段 | 想定到着 |
|------|------|---------|---------|
| A（即応） | 徒歩30分以内 | 徒歩 | 30分以内 |
| B（近距離） | 自転車30分以内 | 自転車・徒歩 | 1時間以内 |
| C（中距離） | 自動車30分以内 | 自動車 | 道路状況次第 |
| D（遠距離） | 上記以外 | 公共交通・自動車 | 不確定 |

> **運用ルール**: Phase 0〜1はA区分の職員だけで回る計画を立てること。C・D区分は「来られたら助かる」の位置づけとする。

### 権限委譲：当直者が「決められる」仕組み

#### 問題の本質

多くの病院で「発災宣言は院長が行う」と規定されている。しかし夜間に院長へ連絡が取れない場合、当直医には宣言権限がなく、初動が凍結する。

#### 事前包括委任の導入

以下の判断権限を、当直責任者（当直医師）に**事前に包括委任**する。

| 判断事項 | 委任範囲 | 事後報告 |
|---------|---------|---------|
| 発災宣言（レベル1〜2） | 当直医師が単独で宣言可 | 宣言後30分以内に院長へ報告 |
| 発災宣言（レベル3） | 当直医師が単独で宣言可 | 直ちに院長・副院長へ連絡試行 |
| 患者避難の開始 | 当直医師 + 夜勤師長の合意 | 避難開始後に院長へ報告 |
| 外部支援の要請 | 当直医師が判断 | 要請後に院長へ報告 |
| EMIS入力 | 当直事務が実施（医師承認不要） | 入力後に情報班長へ報告 |

#### 「判断しない」という判断の禁止

```
【夜間・休日の鉄則】

❶ 迷ったら「上のレベル」で対応する
❷ 連絡がつかない＝委任済みと判断する
❸ 事後報告で許される範囲を明確にしておく
❹ 「院長に確認してから」で初動を止めない
```

### 少人数での初動計画（Phase 0）

夜勤者3〜5名での発災直後の行動を、役割ごとに明確化する。

| 役割 | 担当者 | 最初の5分でやること |
|------|-------|-------------------|
| 指揮 | 当直医師 | 発災宣言、被害の全体把握、外部連絡判断 |
| 患者安全 | 夜勤師長 | 重症患者（呼吸器・透析中）の安全確認 |
| 設備確認 | 施設管理当直 | 電気・水道・ガスの一筆書き簡易点検 |
| 情報・連絡 | 当直事務 | クロノロジー開始、参集連絡の発信 |

> **重要**: 施設管理当直が不在の施設では、当直看護師が設備確認も兼務する。その場合の簡易点検項目は「非常灯の点灯」「水道の通水」「酸素配管の供給圧」の3点に絞る。

## 部署横断型・一筆書きウォークスルー点検

### 目的

災害発生直後または定期訓練時に、施設全体のライフライン3系統（電気・水道・医療ガス）を**部署の境界を越えて**一筆書き方式で巡回点検し、短時間で全系統の稼働状態を災害対策本部に報告する。

### 一筆書き点検とは

通常の点検は各部署が自部門の設備のみを確認するが、一筆書き点検では**1チームが施設全体を途切れなく巡回**し、3系統すべてを横断的に確認する。これにより以下の効果が得られる：

- 部署間の「空白地帯」の見落としを防止
- 点検結果を一元的に災害対策本部へ報告可能
- 複数部署への個別確認を待つ時間を短縮

### 実施手順

#### Step 1: 点検チームの編成

| 役割 | 推奨職種 | 人数 | 担当 |
|------|---------|------|------|
| チームリーダー | 施設課職員 | 1名 | 全体統括・本部連絡 |
| 電気系統担当 | 施設課 or 臨床工学技士 | 1名 | E系列の点検実施 |
| 水道系統担当 | 施設課職員 | 1名 | W系列の点検実施 |
| 医療ガス担当 | 臨床工学技士 | 1名 | G系列の点検実施 |
| 記録係 | 事務職員 | 1名 | クロノロジー記録 |

> **最小構成**: 施設課職員1名 + 臨床工学技士1名の2名でも実施可能（記録はリーダーが兼務）

#### Step 2: 点検テンプレートの準備

- **使用帳票**: `docs/templates/one-stroke-diagram.md`（一筆書き点検テンプレート）
- 印刷済みテンプレートを災害BOXに常備しておくこと
- タブレット端末での電子記録も可

#### Step 3: 巡回ルートの実施

```
【標準巡回ルート（地下→屋上の一筆書き）】

地下階: 受電設備 → 非常用発電機 → 受水槽 → CE室
   │
   ▼
1F: 災害対策本部（中間報告）→ 救急外来 → 給食室
   │
   ▼
各階: 分電盤 → 給水栓 → ガスアウトレット → 区域弁
   │        （各階で3系統を同時確認）
   ▼
重要部門: ICU → 手術室 → 透析室
   │
   ▼
屋上: 高架水槽 → 屋上設備確認
   │
   ▼
災害対策本部: 最終報告
```

#### Step 4: 報告・記録

1. **中間報告**: 1F通過時に災害対策本部へ口頭で「地下階異常なし/あり」を報告
2. **最終報告**: 全巡回完了後、テンプレートを災害対策本部に提出
3. **クロノロジー記載**: 点検開始・完了・異常発見をすべて時系列記録に反映

#### Step 5: 是正対応

| 判定 | 対応 | 担当 |
|------|------|------|
| 全系統正常（◯） | 本部に「全系統正常」を報告 | チームリーダー |
| 要注意（△） | 経過観察リストに追加、次回点検で再確認 | 各系統担当者 |
| 異常あり（×） | 即時に本部報告、応急対応チーム派遣を要請 | チームリーダー |

### 実施タイミング

| 場面 | 実施基準 | 目標所要時間 |
|------|---------|------------|
| 発災直後 | 震度5弱以上 / 大規模停電 / 断水発生時 | 30分以内 |
| 定期訓練 | 年2回以上（うち1回は夜間想定） | 45分以内 |
| 設備更新後 | 系統に関わる設備の更新・改修後 | 60分以内 |

### 訓練での活用

- **初回**: 図面を見ながらルート確認（習熟目的）
- **2回目以降**: 時間を計測し、目標時間内での完了を目指す
- **応用**: 夜間想定（懐中電灯のみ）、一部系統の模擬障害を織り込む

## 動線設計の原則：ゾーニング崩壊の4層構造（物理・動線・時間・合意）

### なぜゾーニングは崩壊するのか

27施設のトリアージ演習フィードバック（アンケート質問3）および札幌東徳洲会病院・茅ヶ崎徳洲会病院・湘南鎌倉総合病院の具体的知見から、ゾーニングが崩壊する原因は**4つの独立した層**に分類されることが判明した。

```
┌─────────────────────────────────────────────────────┐
│         ゾーニング崩壊の4層構造                       │
│                                                       │
│  第1層: 物理（Physics）                              │
│    建物の構造・重力・搬送経路の制約                    │
│    → エレベーター停止で垂直移動が不可能に              │
│    → 札幌東: ブラックアウトでEV・暖房同時停止          │
│                                                       │
│  第2層: 動線（Flow）                                 │
│    人流の交差・視認性・遮蔽すべき経路                  │
│    → 4つの人流が1箇所で衝突                           │
│    → 黒エリアの動線が27施設共通の空白地帯              │
│                                                       │
│  第3層: 時間（Time）                                 │
│    30分後の患者増加・エリア飽和・長期化                │
│    → 再トリアージ基準が標準化されている施設はゼロ      │
│                                                       │
│  第4層: 合意（Consensus）                            │
│    職種間の価値対立・役割の曖昧さ・心理的障壁          │
│    → 医師・看護師・事務が「赤の場所」で揉める          │
│    → 茅ヶ崎:「PTでもトリアージできるのか」            │
│    → 湘南鎌倉: 計画はあるが周知されていない            │
│                                                       │
└─────────────────────────────────────────────────────┘
```

> **従来の「重力・光・心理」の3原則との関係**: 「重力」は第1層（物理）に、「光」は第2層（動線）に、「心理」は第4層（合意）に対応する。第3層（時間）は、アンケート分析の深掘りにより独立した崩壊要因として浮上した。

### 原則1: 物理——下に置け、上に運ぶな

エレベーター停止を前提とし、患者の垂直移動を最小化する。

| エリア | 配置原則 | 理由 |
|-------|---------|------|
| トリアージポスト | 1F（地上階） | 搬入口から最短でアクセス可能 |
| 赤エリア | 搬入口と同一フロア | ストレッチャー搬送の負担を最小化 |
| 黄エリア | 赤エリアの隣接スペース | 状態悪化時の赤への移動を容易に |
| 緑エリア | 自力歩行で到達可能な場所 | 医療資源を赤・黄に集中 |
| 黒エリア | 1Fの目立たない場所 | 搬送負担の最小化 + 視線の遮蔽 |

> **物理の鉄則**: 手術室が上層階にある場合、「赤エリアを手術室の近くに上げる」のではなく「手術機能を1Fに仮設する」ことを検討する。重力には逆らわない。

#### 札幌東徳洲会病院の教訓

2018年北海道胆振東部地震（ブラックアウト）で、札幌東徳洲会病院はコージェネレーションシステム不稼働・非常用コンセント使用不能という事態に直面した。全道295万戸停電によりエレベーター・暖房が同時停止し、**患者を垂直に動かせないうえに低体温で状態が悪化する**二重の物理的制約が発生。寒冷地ではこの物理層の脆弱性が直接的な生命危機に繋がる。

### 原則2: 動線——4つの人流を分離し、黒エリアを遮蔽する

停電時（非常灯のみ）を前提とし、**視認性の確保と視線の遮蔽**を両立する。

#### 4つの動線分離

災害時に1F付近で同時発生する人流を分離する。

```
動線A: 外部搬入（傷病者）     → 搬入口（救急入口）を使用
動線B: 院内避難（入院患者）   → 避難口（正面玄関 or 非常口）を使用
動線C: 家族・帰宅困難者       → 別の入口 or 待機エリアへ誘導
動線D: 黒エリアへの搬送       → 裏口・地下通路など非公開経路
```

**重要**: 動線Aと動線Bが同じ出入口を使う場合、**物理的な時間差**（搬入は正面、避難は裏口）または**物理的な封鎖**（バリケード・ロープ）で分離する。

#### 黒エリアの動線設計

黒エリア（遺体安置場所）の動線は、27施設共通の**空白地帯**である。NotebookLM分析から「黒エリアへの搬送ルートを確保しつつ、他の患者や家族の目に触れさせない動線を確保することは、既存の施設構造上極めて困難」と判明。あまりにセンシティブなテーマであるため演習でも議論が避けられ、最もクリティカルなエリアの動線が未計画のまま残る。

**設計要件**:
1. 黒エリアは1Fの目立たない場所（裏口・地下通路に近接）に配置
2. 搬送ルートは赤・黄エリアの動線と交差しないこと
3. 患者・家族の視線から物理的に遮蔽されること
4. 搬送ルートは図面上でのみ共有し、院内掲示はしない

#### 視認性の事前整備

| 対象 | 整備内容 | 備考 |
|------|---------|------|
| エリア入口 | 蓄光テープ（停電対応）+ 色分けサイン | 災害BOXに常備 |
| 一方通行矢印 | 床面テープ（貼る場所を図面に事前指定） | 柱番号で特定 |
| 黒エリア動線 | 搬送ルートを図面上でのみ共有（掲示しない） | 関係者限定の情報 |

### 原則3: 時間——ゾーニングは動的に変化する

ゾーニングは「最初に引いて終わり」ではない。時間経過に伴う変化を計画する。27施設のアンケートでは、**再トリアージのタイミング基準が標準化されている施設はゼロ**だった。

| 時間帯 | 想定される変化 | 対応 |
|-------|-------------|------|
| 0〜30分 | 一次トリアージ開始 | 初期配置で対応 |
| 30分〜2時間 | 患者数増加、赤エリアの飽和 | 隣接スペースへの拡張（拡張先を事前指定） |
| 2〜6時間 | 二次トリアージ、手術・検査への移行 | 赤エリアの一部を「手術待ち」に転換 |
| 6〜12時間 | 入院患者との統合 | ゾーンと病棟の境界ルールを適用 |
| 12時間〜 | 長期化、スタッフ交代 | ゾーニング解除基準に基づき、通常体制への復帰を判断 |

**時間層の設計要件**:
1. 再トリアージのタイミングを明文化する（例：30分ごと or 状態変化時）
2. 赤エリアの拡張先を事前に指定する
3. 二次トリアージ後の動線を定める（赤→手術室、黄→検査、緑→帰宅）
4. 12時間超の長期化を想定したゾーン再編計画を持つ
5. ゾーニングの解除基準（通常体制への復帰条件）を定める

### 原則4: 合意——職種間ギャップを設計で埋める

#### 職種間の価値対立

27施設のアンケートおよびNotebookLM分析から、ゾーニング配置で医師・看護師・事務の間に構造的な意見の相違が存在する。

| 職種 | 赤エリアの理想位置 | 最重視する価値 |
|------|------------------|-------------|
| 医師 | 処置室・手術室に隣接 | 診療のしやすさ |
| 看護師 | 搬入口に近い場所 | 搬送のしやすさ |
| 事務 | 受付に近い場所 | 記録・家族対応の起点 |

**解決策**: 3職種合同の「図面ワーク」を防災委員会で実施し、各自が理想配置を図面に書き込んだうえで物理層の制約を加え、合意点を形成する。

#### 職種間ギャップの実態：3病院の知見

##### 札幌東徳洲会病院：「誰がトリアージするのか」

アンケートから「誰がトリアージするのか」という素朴かつ深刻な問いが報告された。マニュアル上は「医師」「認定看護師」だが、夜間発災時に医師が到着するまでの空白期間にトリアージポストが無人となるリスクがある。

##### 茅ヶ崎徳洲会病院：「PTでもトリアージできるのか」

「PT（理学療法士）でもトリアージの現場でトリアージできるのか」という問いが報告された。コメディカルの参加基準が曖昧なまま残されている典型例。本フレームワークでは「研修修了者は一次トリアージのみ可」としているが、この基準自体が現場に周知されていない。

##### 湘南鎌倉総合病院：「計画はある、でも周知されていない」

詳細な配置計画（トリアージポスト=医師/認定看護師、誘導係=事務員、記録係=事務員、搬送係=リハビリスタッフ）を持っているにもかかわらず、現場で「私の仕事なのか？」という戸惑いが発生。**仕組みは存在するが、文化として定着していない**典型例。

#### START法と臨床判断の矛盾への対処

アンケートから「START法で緑でも明らかに重症者である場合」「感情をコントロールできない人のトリアージ」への不安が報告されている。

**対処ルール**:

1. START法の判定結果は変えない（速度を維持するため）
2. 臨床的に矛盾を感じた場合は**「要再評価フラグ」**をトリアージタグに付加する
3. フラグ付き患者は、15分以内に医師が再評価する
4. 判定に協力が得られない患者（小児・知的障害者等）は**「個別対応フラグ」**を付加

> これにより、トリアージポストの看護師は「自分の判断で赤に上げていいのか」と迷う必要がなくなる。フラグをつけて緑に送り、医師の再評価を待つ。判断の凍結を防ぐ仕組み。

#### トリアージポストの人員配置

湘南鎌倉総合病院の事例を参考に、トリアージポストの標準配置を定義する。

| 役割 | 担当職種 | 夜間代替 |
|------|---------|---------|
| トリアージ実施者 | 医師 or 認定看護師 | 当直看護師（START法研修修了者） |
| 誘導係 | 事務職員 | 当直事務 |
| 記録係 | 事務職員 | 誘導係が兼務 |
| 搬送係 | リハビリスタッフ | 看護助手 or 応援看護師 |

> **周知の重要性**: 湘南鎌倉はこの配置計画を持っているが、現場では「私の仕事なのか？」という戸惑いが報告されている。計画の存在だけでなく、全職員への周知と訓練での実践が不可欠。

## 組織文化の醸成：善意の判断を守る免責と心理的安全性

### なぜ「文化」が必要なのか

権限委譲と参集計画は「仕組み」です。しかし仕組みだけでは人は動きません。

27施設のアンケートを深掘りすると、権限を持っているはずの当直医が発災宣言をためらう事例が複数報告されました。原因は権限の不在ではなく、**「使ったら罰される」という恐怖**でした。

| 恐怖の類型 | 当直者の心理 | 結果 |
|-----------|------------|------|
| 空振りへの非難 | 「大げさだったと言われる」 | 宣言を先延ばし→初動遅延 |
| 結果責任の追及 | 「あの判断は間違いだった」と言われる | 判断回避→情報収集に逃げる |
| キャリアへの影響 | 「人事評価に響くのでは」 | 院長に確認→連絡待ちで凍結 |

仕組み（権限委譲）＋文化（免責）の両輪がなければ、夜間の初動は動かない。

### 災害時免責条項の導入

#### 免責条項の全文

以下の免責条項を文書化し、**院長の署名**を付して正式な院内方針とする。

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   【災害時免責条項】                                     │
│                                                         │
│   1. 災害発生時またはその疑いがある状況において、         │
│      当直者が善意に基づき行った判断・行動は、             │
│      その結果が最善でなかった場合であっても、             │
│      懲戒処分および人事評価上の不利益の対象としない。     │
│                                                         │
│   2. 「空振り」（結果的に大規模災害でなかった場合の       │
│      発災宣言や避難指示）は、組織として正しい判断と       │
│      位置づけ、事後の非難を禁止する。                     │
│                                                         │
│   3. 本条項は、故意または重大な過失による行為には         │
│      適用しない。                                        │
│                                                         │
│   制定日:＿＿＿＿年＿＿月＿＿日                           │
│   病院長署名:＿＿＿＿＿＿＿＿＿                           │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

#### 院長署名が不可欠な理由

| 理由 | 説明 |
|------|------|
| 組織の意思表示 | 「個人の善意に頼る」のではなく「組織として守る」という公式宣言 |
| 心理的安全性 | 「院長がサインした方針に従った」と言えることが、当直者の心理的盾になる |
| 事後検証での根拠 | 院内事故調査委員会等で「組織方針に基づいた行動」と位置づけられる |
| 空振り文化の確立 | トップが署名することで「空振りを罰しない」が組織全体のメッセージになる |

### 「空振りは正しい」の制度化

#### 航空業界のGo-Around原則

航空業界では、パイロットが着陸に不安を感じた場合、着陸をやり直す「Go-Around」が奨励されています。結果的に着陸できたかもしれなくても、Go-Aroundの判断は称賛され、記録には「安全側の判断」として残ります。罰する航空会社はありません。

**病院の発災宣言も同じです。**

- 震度5弱以上で発災宣言を出し、結果的に被害が軽微だった → **正しい判断**
- 停電が発生し避難準備を指示したが、5分後に復旧した → **正しい判断**
- 異常な揺れを感じて緊急連絡網を発動したが、震度3だった → **正しい判断**

#### 空振り報告の運用

空振りを「失敗」ではなく「実績」として記録する仕組みを設ける。

| 項目 | 運用方法 |
|------|---------|
| 記録 | クロノロジーに「発災宣言（レベル○）→結果的に解除」として記録 |
| 報告 | 事後報告で十分。「なぜ宣言したか」ではなく「何を根拠に判断したか」を記録 |
| 評価 | 防災委員会で「適切な安全側判断」として肯定的に言及 |
| 共有 | 全職員向けに「この判断は正しかった」とフィードバック |

### 心理的安全性の3層構造

免責条項は、組織の心理的安全性を支える3層構造の一部として機能する。

```
第3層: 訓練と習熟
  │    「やったことがあるから、できる」
  │    → 夜間想定訓練、抜き打ち訓練
  │
第2層: 権限と仕組み
  │    「権限があるから、判断していい」
  │    → 事前包括委任、代行順位
  │
第1層: 免責と文化（土台）
       「守られているから、怖くない」
       → 免責条項、空振り奨励、院長署名
```

**第1層（免責と文化）が欠けていると、第2層・第3層がどれだけ整備されても機能しない。** 権限があっても怖くて使えず、訓練していても本番で躊躇する。

### 事後検証（After Action Review）のルール

災害対応後の振り返りは必須だが、「犯人捜し」になった瞬間、次の災害で誰も動けなくなる。

#### 事後検証の原則

1. **目的**: 「次にどうすればもっと良くなるか」を見つけること（改善志向）
2. **禁止**: 「誰が悪かったか」を追及すること（懲罰志向）
3. **前提**: 全員が善意で最善を尽くしていたと推定する
4. **対象**: 個人の判断ではなく、仕組みと情報の問題を分析する

#### 事後検証のフォーマット

| 観点 | 問い | NG例（懲罰的） |
|------|------|---------------|
| 情報 | 判断時点で十分な情報があったか？ | 「なぜ○○を確認しなかったのか」 |
| 仕組み | 情報が当直者に届く仕組みはあったか？ | 「当直医の判断が遅かった」 |
| ツール | アクションカード・設備は機能したか？ | 「使い方を知らなかったのか」 |
| 改善 | 次回に向けて何を変えるべきか？ | 「二度とこのような失態がないように」 |

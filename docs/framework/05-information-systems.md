# 軸5: 情報システム（システムダウン対応）

## 概要

電子カルテ、医事システム、検査システムなど、現代の病院は多くの情報システムに依存しています。システムダウン時の紙運用への切替と業務継続手順を準備しておく必要があります。

## リスク評価

### 主要システムと途絶時影響

| システム | 停止時の影響 | 代替手段 | 復旧優先度 |
|---------|------------|---------|----------|
| 電子カルテ | 診療情報参照不可 | 紙カルテ・サマリー | 最優先 |
| オーダリング | 指示伝達不可 | 伝票運用 | 最優先 |
| 医事システム | 会計処理不可 | 後日精算 | 中 |
| 検査システム | 結果参照不可 | 電話報告 | 優先 |
| 薬剤システム | 調剤支援不可 | 手書き処方 | 優先 |
| 画像システム | 画像参照不可 | フィルム出力 | 優先 |
| 看護システム | 記録不可 | 紙記録 | 中 |

## 対応フロー

### Phase 1: 初動（0〜15分）

```
システムダウン発生
    ↓
原因確認（停電 or システム障害）
    ↓
[停電の場合]          [システム障害の場合]
    ↓                    ↓
電源復旧待ち        システム担当連絡
    ↓                    ↓
UPS時間内に         ベンダー連絡
バックアップ取得      復旧見込み確認
```

### Phase 2: 紙運用切替（15分〜）

1. **紙運用開始宣言**
   - 院内放送（可能なら）
   - 各部署への通知

2. **紙伝票・用紙の配布**
   - 処方箋用紙
   - 検査依頼伝票
   - 看護記録用紙

3. **患者情報の確認手段**
   - 直近プリントアウト
   - 患者サマリー
   - お薬手帳

### Phase 3: 紙運用継続

- 定時での情報集約
- 二重入力の準備（復旧後用）
- 重要情報の複写保管

## 紙運用時の業務フロー

### 外来診療

```
患者受付（紙受付票）
    ↓
診察（紙カルテ or メモ）
    ↓
処方（手書き処方箋）
    ↓
会計（後日精算 or 概算）
```

### 入院病棟

```
指示（口頭 + 指示簿）
    ↓
実施（紙記録）
    ↓
定時報告（電話 or 伝令）
```

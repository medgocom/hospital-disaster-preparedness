# 停電3秒後、あなたの病院の「脳」と「神経」は同時に死ぬ

**〜サーバー室の熱暴走、ハブの電源喪失、PBXの沈黙——27施設が見落とした「デジタル沈黙」の恐怖〜**

---

## はじめに：病院には2つの神経系がある

人間の身体には、脳と神経がある。

病院にも、2つの神経系がある。

1つ目は**人間の神経系**。医師が判断し、看護師が動き、事務が連絡する。声と身体による情報伝達。

2つ目は**デジタルの神経系**。電子カルテ（脳）、ネットワークハブ（神経）、PHS・PBX（声帯）。電気信号による情報伝達。

現代の病院は、この2つの神経系が協働して動いている。

**停電すると、デジタルの神経系は3秒で死にます。**

UPSが起動するまでの数秒間、すべてが暗転します。UPSが起動しても、**非常用電源に接続されていない機器は復旧しない。** そして、多くの中間ネットワーク機器は非常用電源に接続されていない。

電子カルテの端末は真っ暗。PHSは沈黙。ナースコールの挙動は不明。院内放送は使えない。

**これが「デジタル沈黙」だ。**

---

## 第1章：サーバー室は「脳」である

### 脳が死ねば、手足は動かない

電子カルテのサーバーは、病院のあらゆる診療情報を格納している。患者の病名、投薬履歴、検査結果、手術記録。**150人の入院患者の全情報が、この部屋の中にある。**

サーバー室の保護は、多くの施設で意識されている。非常用電源への接続、UPSの設置。ここまではできている施設が多い。

**問題は、電源以外の脆弱性だ。**

### 熱暴走——電源があってもサーバーは死ぬ

サーバーは大量の熱を発生します。通常は空調で冷却していますが、**空調が非常用電源に接続されていなければ、停電と同時にサーバー室の温度が上昇し始めます。**

夏季の場合：

| 経過時間 | サーバー室温度 | サーバーの状態 |
|---------|-------------|-------------|
| 0分 | 25℃（通常） | 正常稼働 |
| 30分 | 30〜35℃ | 警告温度 |
| 1時間 | 35〜40℃ | パフォーマンス低下 |
| 2時間 | 40℃超 | 自動シャットダウンまたは故障 |

**非常用電源でサーバーが動いていても、冷却できなければ2時間で「脳死」する。**

### 27施設のデータ

| 項目 | 施設数 |
|------|-------|
| サーバー室が非常用電源に接続 | 18施設 |
| サーバー室空調が非常用電源に接続 | 6施設 |
| サーバー室の停電時冷却手段がある | 3施設 |

サーバー本体の電源は意識されているが、**冷却の電源は意識されていない。** これが典型的な盲点だ。

## 第2章：ネットワークハブは「神経」である

### 脳が生きていても、神経が断たれれば動けない

サーバーが動いていても、各端末からサーバーにアクセスできなければ意味がない。サーバーと端末を繋ぐのがネットワークスイッチ（ハブ）だ。

このハブは、多くの場合**各フロアのEPS（電気シャフト）やMDF（配線盤室）**に設置されている。廊下の隅の小さな扉の中。目立たない場所だ。

**そして、ほとんどの施設で非常用電源に接続されていない。**

### 停電時に起こること

```
【通常時】
  端末 ←→ フロアハブ ←→ 基幹スイッチ ←→ サーバー
           ↑ 通常電源      ↑ 非常電源      ↑ 非常電源

【停電時】
  端末 ←×→ フロアハブ（停止）  基幹スイッチ ←→ サーバー
           ↑ 電源喪失            ↑ 非常電源     ↑ 非常電源

  結果：サーバーは動いているが、端末からアクセスできない
```

**サーバー（脳）は生きている。しかし神経（ハブ）が断たれたため、手足（端末）は動かない。**

### 「電子カルテが使えない」の本当の原因

停電時に「電子カルテが使えない」と報告された場合、原因は3つ考えられます。

1. サーバー自体がダウンした
2. ネットワークハブが停電で落ちた
3. 端末の電源が落ちた

**多くの場合、原因は2番目です。** サーバーは非常用電源で動いている。端末もUPS付きのものがある。しかし、その間を繋ぐハブが死んでいる。

## 第3章：PBXの沈黙——院内電話の全滅

### 電話の「脳」も熱で死ぬ

PBX（Private Branch Exchange / 構内交換機）は、院内の電話交換を司る装置です。内線電話、外線発着信、ナースコール連携——すべてがPBXを経由している。

PBXは多くの施設で非常用電源に接続されている。**しかし、PBXも発熱する機器だ。**

空調が止まった状態でPBXが稼働し続けると、夏季には数時間で過熱し、**熱停止（thermal shutdown）**する可能性がある。

PBXが停止すると：

- 院内の内線電話が全滅
- 外線の発着信が不可
- ナースコールのPBX連携機能が停止
- PHSの内線通話が不可

### PHS基地局の電源問題

PHSは、院内に設置された基地局を経由して通話します。この基地局は各フロアの天井裏に設置されていることが多く、**非常用電源に接続されていない施設が大半だ。**

PBXが動いていてもPHS基地局が停電で落ちれば、院内PHSは使用不可になります。

27施設の結果：

| 項目 | 施設数 |
|------|-------|
| PBXが非常用電源に接続 | 20施設 |
| PBXの冷却対策がある | 4施設 |
| PHS基地局が非常用電源に接続 | 6施設 |
| PHS基地局の電源状況を把握 | 8施設 |

**PBX本体は保護されているが、PHS基地局は保護されていない。** 結果として、PBXが動いていても院内PHSは使えない。

## 第4章：紙に戻る覚悟——「デジタル沈黙」への現実的対応

### デジタルが死んだら、紙で戦う

デジタル沈黙が発生した場合、病院は**紙とペンと口頭指示**で診療を継続しなければならない。

これは「昔に戻る」だけのことではない。**電子カルテに慣れた世代が、紙運用を経験したことがない**という現実がある。

### 紙運用に必要な事前準備

| 準備項目 | 具体的内容 | 更新頻度 |
|---------|----------|---------|
| 入院患者一覧 | 氏名・病名・主治医・主な投薬 | 毎日印刷 |
| 処方箋用紙 | 手書き処方用の白紙伝票 | 常時備蓄 |
| 検査依頼伝票 | 手書き検査依頼用 | 常時備蓄 |
| 看護記録用紙 | 時系列記録用 | 常時備蓄 |
| 患者サマリー | 主要患者の診療要約 | 週1回印刷 |
| 採用薬一覧 | 院内採用薬の名称・用法・用量 | 月1回印刷 |

### 「入院患者一覧」の毎日印刷が生死を分ける

最も重要なのは**入院患者一覧の毎日印刷**だ。

停電で電子カルテが使えなくなった瞬間、「この患者は何の薬を飲んでいるか」「アレルギーはあるか」「DNARの指示はあるか」——これらの情報に、紙でしかアクセスできなくなります。

毎日印刷していれば、最悪でも昨日の情報がある。印刷していなければ、情報はゼロだ。

## 第5章：SNSという「第三の神経系」

### 院内ネットワークが死んでも、携帯電話のデータ通信は生きている

ここで一つの希望がある。

院内のネットワーク（有線LAN、Wi-Fi、PHS）が全滅しても、**携帯電話のモバイルデータ通信が生きていれば、LINEやメールが使えます。**

携帯基地局が生きている限り（停電でも基地局には非常用電源がある）、スマートフォンでの連絡は可能だ。

### LINE連絡網の構築

災害時にLINEを「第三の神経系」として活用するための準備：

| グループ | メンバー | 用途 |
|---------|--------|------|
| 【災害対策本部】 | 院長・副院長・看護部長・事務長 | 経営判断 |
| 【○○病棟】 | 各病棟スタッフ | 現場情報共有 |
| 【全職員】 | 全職員 | 一斉周知・安否確認 |

### バッテリー管理が命綱

ただし、LINEが使えるのは**スマートフォンのバッテリーが持つ間だけ**だ。

停電が長期化した場合のバッテリー節約ルール：

1. 画面の明るさを最低に
2. Wi-Fi・Bluetoothをオフ（モバイルデータのみ）
3. 写真・動画の送信は最小限（テキストのみ推奨）
4. 充電器・モバイルバッテリーの災害BOX配備

## 第6章：SNSグループの公式バックアップ化——プライバシーを守る「災害時運用規約」

### 個人ツールを病院業務に使う矛盾

前章でLINEを「第三の神経系」として紹介しました。しかし、現実には大きな問題がある。

**LINEは個人のツールだ。**

個人のスマートフォンに入っている個人のアプリを、病院業務に使う。患者情報が流れるリスク。プライベートと業務の境界が曖昧になる。職員の心理的抵抗。

これらの問題を放置したまま「災害時はLINEを使え」と言っても、機能しない。必要なのは、**「災害時SNS運用規約」**だ。

### 災害時SNS運用規約（ひな形）

以下は、災害時にLINE等のSNSを公式のバックアップ通信手段として使用するための運用規約のひな形です。防災委員会で承認し、全職員に周知すべきだ。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   【○○病院 災害時SNS運用規約】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

第1条（目的）
本規約は、災害により院内通信（PHS・内線電話・
院内放送）が使用不能となった場合に限り、
LINE等のSNSを公式のバックアップ通信手段として
使用するための運用ルールを定める。

第2条（発動条件）
本規約は、以下のいずれかに該当する場合に
災害対策本部長が発動する。
 (1) 院内PHS・内線電話が全面停止した場合
 (2) 災害対策本部が通信途絶を宣言した場合
 (3) 上記に準じる通信障害が発生した場合

第3条（使用するSNS）
原則としてLINEを使用する。
事前に以下のグループを作成・維持する。
 ・【災害対策本部】（院長・副院長・部長級）
 ・【○○病棟】（各部署別）
 ・【全職員連絡】（一斉周知・安否確認用）

第4条（禁止事項——患者情報の保護）
以下の情報をSNSに投稿してはならない。
 (1) 患者の氏名
 (2) 患者のカルテ番号・生年月日
 (3) 患者の顔が写った写真・動画
 (4) 診断名・検査結果などの診療情報

第5条（許可される情報）
以下の情報は投稿して差し支えない。
 (1) 施設の被害状況（写真含む）
 (2) ライフラインの状態（停電・断水等）
 (3) 「○階病棟、患者全員の安全確認済み」
     等の匿名化された状況報告
 (4) 出勤要請・安否確認
 (5) 物資の過不足情報

第6条（患者を特定せずに伝える技術）
患者の状態を伝える必要がある場合、
以下の匿名化ルールに従う。
 ・「3F-201号室の方」ではなく
   「3F重症1名、バイタル不安定」
 ・「○○さんの透析」ではなく
   「透析患者1名、継続要否判断必要」
 ・数と状態で伝え、個人を特定しない

第7条（写真・動画の取り扱い）
 (1) 施設の被害写真は積極的に共有する
   （状況把握に極めて有効）
 (2) 患者が映り込む可能性がある場合は
     撮影しない
 (3) 撮影する場合は患者が映らない角度で

第8条（バッテリー管理）
 (1) 通信はテキストを原則とし、
     写真・動画の送信は最小限にとどめる
 (2) スマートフォンの省電力設定を推奨
 (3) 災害BOXにモバイルバッテリーを配備

第9条（平時の管理）
 (1) グループメンバーは年2回更新する
     （人事異動・退職者の反映）
 (2) 日常業務での使用は禁止する
 (3) 年1回、災害訓練時にSNS連絡訓練を
     実施する

第10条（規約の発効）
本規約は防災委員会の承認をもって発効する。
改定は防災委員会の決議による。

                    承認日：　　年　月　日
                    病院長署名：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### なぜ「規約」が必要なのか

| 理由 | 説明 |
|------|------|
| 心理的障壁の除去 | 「個人のLINEを業務に使っていいのか」という不安を解消 |
| 患者情報保護 | 何を送っていいか・いけないかを明確にし、情報漏洩を防ぐ |
| 事後の説明責任 | 「規約に基づいた運用だった」と説明できる |
| 平時と災害時の境界 | 発動条件を明確にし、日常的な業務利用を防止 |

### 導入のポイント

1. **院長署名を必ず取る** — 免責条項と同じ。署名がなければ「お気持ち表明」で終わる
2. **訓練で1回使う** — 規約を配っただけでは誰も使えない。訓練でLINEグループに実際に投稿する体験が必要
3. **年2回のメンバー更新** — 退職者のグループ残留が最大のセキュリティリスク

## 第7章：サイバー攻撃という「もう一つの停電」

### ランサムウェアは「デジタル停電」である

近年、病院へのランサムウェア攻撃が増加している。ランサムウェアに感染すると、電子カルテを含む全システムが暗号化され、使用不能になります。

**これは、物理的な停電がなくても発生する「デジタル停電」だ。**

物理的停電との違い：

| | 物理的停電 | ランサムウェア |
|--|---------|-------------|
| 原因 | 地震・台風・設備故障 | サイバー攻撃 |
| 影響範囲 | 電力依存設備 | 全情報システム |
| 復旧見込み | 数時間〜数日 | 数日〜数週間 |
| データ損失 | なし（通常） | 暗号化による喪失リスク |

### 共通する対策

物理的停電とランサムウェアの対策には共通点がある。

- **紙運用への切替手順** → 両方に有効
- **定期バックアップ（オフライン保管）** → 特にランサムウェアに有効
- **入院患者一覧の毎日印刷** → 両方に有効

つまり、**災害対策として紙運用を訓練しておけば、サイバー攻撃にも対応できる。** 一粒で二度の備えになります。

---

## おわりに：デジタル沈黙を想定せよ

現代の病院は、デジタルの神経系なしでは動けません。電子カルテ、PHS、ナースコール、院内放送。すべてが電気で動き、ネットワークで繋がっている。

**そして、そのすべてが停電3秒で止まる可能性があります。**

「デジタル沈黙」——電子カルテが見えない、電話が通じない、ナースコールが鳴らない。この状態が、あなたの病院で本当に起きたとき、何ができますか？

今日できることは4つだ。

1. **ネットワークハブの電源を確認する**（非常用電源に接続されているか）
2. **PHS基地局の電源を確認する**（停電で院内PHSが使えなくなるか）
3. **入院患者一覧を今日から毎日印刷する**（最も安価で最も効果的な対策）
4. **紙運用訓練を1回実施する**（やってみなければ、できるかどうか分からない）

**デジタル沈黙は、必ず来ます。** 来たときに「想定していなかった」と言わないために。
